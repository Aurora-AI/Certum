# WORKFLOW OPERACIONAL AURORA

Para garantir a excelência, siga este fluxo para novas funcionalidades:

1.  **FASE 1: CONCEPÇÃO (Agente: ARCHITECT)**
    - Usuário solicita feature.
    - Arquiteto analisa Constituição e Doc de Bibliotecas.
    - Arquiteto gera `PLAN.md`.

2.  **FASE 2: CONSTRUÇÃO (Agente: BUILDER)**
    - Builder lê `PLAN.md`.
    - Builder implementa código em passos pequenos.
    - Builder aplica estilos e animações.

3.  **FASE 3: AUDITORIA (Agente: REVIEWER)**
    - Builder submete o código.
    - Reviewer analisa contra a `constitution.md`.
    - Se aprovado -> Commit.
    - Se reprovado -> Volta para Fase 2 com instruções.
